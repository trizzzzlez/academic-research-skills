# Revision Response Template — 修訂回應模板

本模板幫助作者系統性回應所有審查意見。格式為 Reviewer Comment → Author Response → Changes Made（R→A→C 格式）。

---

## 使用說明

1. 每一項 reviewer 的意見都必須回應，不可跳過
2. 如果不同意某項意見，必須說明理由（不是簡單的「不同意」）
3. 修改標示：在修訂稿中以追蹤修訂或顏色標示所有變更
4. 頁碼對照：提供原稿和修訂稿的頁碼對照

---

## 回應的品質標準

### 好的回應特徵

1. **直接**：不迴避問題，正面回應每一項意見
2. **具體**：指出修改的確切位置和內容
3. **有證據**：如果不同意 reviewer，提供文獻或數據支撐
4. **禮貌**：感謝 reviewer 的回饋，即使不同意
5. **完整**：沒有遺漏任何一項 reviewer 意見

### 壞的回應特徵

1. **敷衍**：「已修改」（但不說改了什麼）
2. **迴避**：對困難的問題避而不答
3. **防禦**：「Reviewer 誤解了我的論文」（但不解釋為什麼）
4. **過度承諾**：承認所有問題但不提供解決方案
5. **缺少標示**：修改了但沒有標示位置，讓 reviewer 找不到

### 不同意 Reviewer 的正確方式

```markdown
# 正確方式
> Reviewer: 建議使用 X 方法替代 Y 方法

**Author Response**: 我們感謝 Reviewer 的建議。我們選擇 Y 方法而非 X 方法的理由如下：
1. Y 方法在處理 [特定類型] 的數據時有更好的表現（Smith et al., 2023）
2. 我們的研究設計中，X 方法的前提假設（如 [假設]）並不滿足
3. 然而，我們在修訂稿中新增了使用 X 方法的 robustness check（附錄 B），結果一致。

**Changes Made**: 新增附錄 B（p.25-26），包含 X 方法的 robustness check 結果。在方法論章節（p.9, para.3）新增了選擇 Y 方法的理由說明。

# 錯誤方式
> Reviewer: 建議使用 X 方法替代 Y 方法

**Author Response**: 我們不同意。Y 方法是合適的。
```
